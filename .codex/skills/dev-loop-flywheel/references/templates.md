# 开发循环记忆模板

只有在写入或更新 `.ai/memory/**` 时才读取本文件。

## 目录

- 可行动性测试
- 模式
- Keep 教训
- Avoid 教训
- ID 和去重

## 可行动性测试

每条模式或教训都必须能补全这句话：

```text
下次遇到 <触发情境> 时，应该做 <具体行为 A>，而不是 <行为 B>，因为 <本轮证据>。
```

如果内容只是“更仔细”“加强沟通”“提升质量”这类空泛建议，必须拒绝或重写。

## 模式

```md
## P-YYYYMMDD-001: <短名称>

日期:
- YYYY-MM-DD

状态:
- active | superseded

触发:
- <什么时候应该考虑这个模式>

模式:
- <可复用结构或重复机制>

规则:
- <未来必须执行的具体行为>

适用范围:
- <路径、任务类型、skill、spec 或 workflow>

来源:
- <review finding、用户纠正、run id 或 task id>

证据:
- <本轮的短证据>

关联 ID:
- <可选的 memory 或 finding id>
```

## ID 和去重

- 生成 ID 时，先扫描同日期、同前缀的已有条目，再把最大数字后缀加 1。例如已有 `P-20260701-003`，下一个就是 `P-20260701-004`。
- 去重时，把规则文本转成小写、去掉多余空格、忽略纯标点差异，并比较核心动词/对象/动作。
- 如果相同规则已经存在，且本轮没有新增行为，不要重复追加。
- 如果相同规则有新证据，尽量在旧条目下追加简短 `Update`。
- 如果规则发生变化，新增条目并写 `Supersedes: <old-id>`。
- 如果修改旧条目容易造成 churn，就新增条目，并在 `关联 ID` 里引用旧 ID。
- 除非结构化 reviewer 输出明确要求更多，否则每轮最多推断 3 条。
